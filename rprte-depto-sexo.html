NOTA SOBRE EL USO DE GO TO
Su uso esta segmentado exclusivamente para manejar el flujo
de ejecucion dentro del estado de error. No interfiere en el
flujo de la logica de negocio, el cual respeta la
programacion estructurada y la ejecucion TOP-DOWN.
MANEJO DE ERRORES
DECLARATIVES: Captura errores de E/S en archivo de salida
WHENEVER SQLERROR: Captura errores SQL
En ambos casos se deriva a 2300-INVOCAR-RUTINA-ERROR que:
1. Carga WS-ERROR con el contexto del error
2. Llama a RUTERRBA via CALL
3. Activa PGM-FIN para encauzar el flujo al cierre
WS-OPERACION refleja el estado en curso al momento del error
garantizando un cierre ordenado en 3000-FINAL
ESTADO DE LA APLICACION
El programa se estructura en torno a un estado general definido
por niveles 88 de WS-OPERACION. Cada bloque de código relevante
setea el estado x medio de SET. Esto permite:
1. Identificar claramente la operación en curso en cualquier
punto.
2. Manejar errores de forma centralizada en
2300-INVOCAR-RUTINA-ERROR utilizando el estado para determinar
el punto de retorno adecuado y garantizar un cierre ordenado.
3. Mantener la lógica de negocio estructurada y legible,
evitando el uso indiscriminado de GO TO, limitándolo
exclusivamente al manejo y control por medio de EVALUATE TRUE.
CENTRADO DE SUBTITULO DEPARTAMENTO
Centra un texto dentro de un campo de 70 caracteres.
1. TRIM elimina espacios finales del texto
2. LENGTH calcula la longitud real del texto
3. PADDING = (70 - longitud) / 2 = espacios a la izquierda
4. Mueve el texto a la posicion calculada dentro del campo

LAYOUT DE SALIDA PARA REPORTE RPTCLI01
DESCRIPCIÓN
Define las líneas de impresión del reporte de empleados
agrupados por departamento y sexo. El archivo de salida tiene
LRECL=72 y es usa control de página mediante LINAGE.
Incluye bordes, títulos, cabeceras, líneas de detalle,
subtotales por sexo, totales por departamento y total general.
CAMPOS PRINCIPALES
- RPT-BORDE-ASTERISCO : Línea de asteriscos borde superior.
- RPT-TITULO : Línea de título con fecha y página.
* RPT-TIT-FECHA : Fecha de ejecución (AAAA-MM-DD).
* RPT-TIT-PAGINA : Número de página actual.
- RPT-BORDE-GUION : Borde de guiones que separa secciones.
- RPT-CAB-DEPT : Línea que muestra el nombre del depto.
* RPT-CDP-TEXTO : Texto con el nombre del depto.
- RPT-CAB-SEXO : Línea que muestra el sexo actual.
* RPT-CSX-DESC-SEXO : Descripción del sexo.
- RPT-COLUMNAS : Cabecera de columnas.
- RPT-DETALLE : Fila de detalle de cada empleado.
* RPT-DET-EMPNO : Número de empleado (6 dígitos).
* RPT-DET-NOMBRE : Primer nombre (12 caracteres).
* RPT-DET-APELLIDO : Apellido (15 caracteres).
* RPT-DET-SALARIO : Salario con formato monetario.
- RPT-BORDE-IGUAL : Borde de igual separa subtotales.
- RPT-SUBTOTAL-SEXO : Subtotal por sexo dentro del depto.
* RPT-SSX-SEXO : Sexo (F/M) del subtotal.
* RPT-SSX-CANTIDAD : Cantidad de empleados de ese sexo.
* RPT-SSX-PROMEDIO : Salario promedio del grupo.
- RPT-TOTAL-DEPT : Total del departamento.
* RPT-TDP-DEPTNO : Número de depto (3 dígitos).
* RPT-TDP-CANTIDAD : Total empleados del depto.
* RPT-TDP-PROMEDIO : Salario promedio del depto.
- RPT-TOTAL-GRAL : Total general de todo el reporte.
* RPT-TGR-CANTIDAD : Suma total de empleados.
* RPT-TGR-PROMEDIO : Salario promedio general.
- RPT-BARRA-INCLINADA : Línea de / inclinadas.
- RPT-LINEA-BLANCA : Línea en blanco con bordes laterales.
NOTAS DE IMPLEMENTACIÓN
1. Todos los bordes usan el carácter '|' en la columna 1 y 72.
3. Los campos FILLER se usan para espaciado fijo.

DESCRIPCIÓN
Estructura generada automáticamente por el comando DCLGEN
para declarar host variables COBOL que reflejan la estructura
de la tabla DEPT de DB2. Los nombres de campos llevan el
prefijo WS-.

- WS-DEPTNO : Nº de depto. (clave primaria, PIC X(3)).
- WS-DEPTNAME : Nombre del depto.
* WS-DEPTNAME-LEN : Longitud real (COMP-5).
* WS-DEPTNAME-TEXT: Texto del nombre.
- WS-MGRNO : Nº de empleado
- WS-ADMRDEPT : Departamento administrativo (PIC X(3)).
- WS-LOCATION : Ubicación geográfica (PIC X(16)).
(Total de 5 columnas)
NOTAS DE IMPLEMENTACIÓN
1. No modificar manualmente este copybook.
2. Si la tabla cambia, regenerar con DCLGEN (ACTION(REPLACE)).

ESTRUCTURA COMPARTIDA PARA MANEJO CENTRALIZADO DE ERRORES
DESCRIPCIÓN
Define la estructura de datos utilizada para transportar
información de errores desde los programas batch hacia la
rutina RUTERRBA. Consolida datos del programa origen,
códigos SQL, tipo de error y contexto de ejecución.
CAMPOS
- WS-ERR-PROGRAMA: Nombre PGM Main.
- WS-ERR-FS: File status de 2 caracteres (si aplica)
- WS-ERR-SQLCA: SQLCA completo capturado en excepción
- WS-TIPO-ERROR: Uno de los valores de condición definidos
NOTAS DE IMPLEMENTACIÓN
1. La redefinición SQLCA extrae el SQLCODE para análisis
2. Todos los campos se inicializan con SPACES/ZEROS
3. RUTERRBA procesa y displaya la información.
